# SHIBUMI · התקנת Backend

מדריך מלא להפעיל את ה-Backend של האתר — כך שפניות מטופס יצירת הקשר
יגיעו אליך אוטומטית במייל ויישמרו ב-Google Sheet.

**זמן: כ-5 דקות. עלות: 0 ש"ח. ללא שרת, ללא תחזוקה.**

---

## איך זה עובד

```
גולש מהטלפון/מחשב  →  טופס "יצירת קשר" באתר  →  Apps Script (Google)
                                                 ├── שורה חדשה ב-Google Sheet
                                                 ├── מייל אליך
                                                 └── תשובה לאתר: "התקבל"

מערכת הניהול (/admin/leads.html)  →  Apps Script  →  Google Sheet → תצוגה בטבלה
```

---

## שלב 1 · יצירת ה-Script

1. פתח את [script.google.com](https://script.google.com) עם חשבון ה-Gmail שממנו תרצה שישלחו ההתראות.
2. לחץ על **New project**.
3. מחק את כל הקוד שיש ב-`Code.gs` (ברירת המחדל).
4. פתח את הקובץ `SHIBUMI_backend.gs` שנמצא ברפוזיטורי המקומי שלך.
5. העתק את *כל* התוכן והדבק ב-`Code.gs` של Apps Script.
6. בראש הקובץ, ב-`const CONFIG = { ... }`, עדכן:
   - `BROKER_EMAIL` — המייל שלך (לקבלת התראות).
   - `ADMIN_TOKEN` — מחרוזת אקראית ארוכה (24+ תווים). דוגמה:
     ```
     'shibumi-4f8a2c9d-b3e7-41a2-9f6e-8c5d3a7e9b14'
     ```
     **שמור אותה בצד — תצטרך אותה בשלב 3.**
7. Ctrl+S לשמור. תן לפרויקט שם: `SHIBUMI Backend`.

---

## שלב 2 · פריסה כ-Web App

1. בתפריט העליון של Apps Script, בחר את הפונקציה **`setupShibumiBackend`**
   (מהרשימה הנפתחת ליד כפתור Run) ולחץ **Run**.
   - בפעם הראשונה: יבקש הרשאות → **Review permissions** → בחר את החשבון
     → **Advanced** → **Go to SHIBUMI Backend (unsafe)** → **Allow**.
   - זו התראה סטנדרטית של Google על סקריפטים של משתמש פרטי — זה בסדר, זה הסקריפט שלך.
2. אחרי שזה רץ, תראה בלוג (**View → Logs**) את הקישור לגיליון שנוצר.
   שמור אותו — זה ה-Google Sheet שבו יופיעו כל הפניות.
3. עכשיו לחץ על **Deploy** (פינה ימנית עליונה) → **New deployment**.
4. לחץ על סמל הגלגל (⚙) ליד "Select type" → בחר **Web app**.
5. מלא כך:
   - **Description:** `shibumi v1`
   - **Execute as:** `Me (your@email)`
   - **Who has access:** `Anyone`
6. לחץ **Deploy**. ייתכן שיבקש שוב אישור — אשר.
7. **העתק את ה-Web app URL** — נראה ככה:
   ```
   https://script.google.com/macros/s/AKfy...../exec
   ```

## שלב 3 · חיבור האתר

1. פתח את הקובץ `site/js/config.js`.
2. מלא את שני הערכים:
   ```js
   window.SHIBUMI_CONFIG = {
     BACKEND_URL: "https://script.google.com/macros/s/AKfy...../exec",
     ADMIN_TOKEN: "shibumi-4f8a2c9d-b3e7-41a2-9f6e-8c5d3a7e9b14",
     PHONE: "050-733-0090",
   };
   ```
   ה-`ADMIN_TOKEN` **חייב** להיות *בדיוק אותה מחרוזת* שכתבת ב-`CONFIG.ADMIN_TOKEN`
   ב-`SHIBUMI_backend.gs`.
3. שמור את הקובץ.

---

## שלב 4 · בדיקה

1. פתח את האתר: `site/contact.html`.
2. מלא את הטופס (שם + טלפון + הודעה) ושלח.
3. תוך שניות:
   - בתיבת המייל שלך יגיע "פנייה חדשה באתר · [שם]".
   - ב-Google Sheet תופיע שורה חדשה.
4. עכשיו פתח את `site/admin/leads.html` והיכנס.
   - בשורת הסטטוס למעלה יופיע: **● מחובר ל-Backend**.
   - הפנייה שלך מופיעה בטבלה.

## מה עובד מול ה-Backend

| פונקציה                          | מקומי | מחובר ל-Backend |
|----------------------------------|:-----:|:----------------:|
| טופס יצירת קשר מהאתר             |  ✓   | ✓ + מייל + גיליון |
| צפייה בפניות במערכת הניהול       |  ✓   | ✓ (מכל מכשיר)    |
| מחיקת פניות                       |  ✓   | ✓                 |
| עריכת נכסים (כותרת/מחיר/סטטוס)   |  ✓   | מקומי בלבד *     |
| הסתרת נכסים                       |  ✓   | מקומי בלבד *     |
| סיסמת מנהל                        |  ✓   | מקומי בלבד *     |

*לנכסים: מכיוון שמדובר בקטלוג של מספר עשרות נכסים שמתעדכן לעתים רחוקות,
עריכה מקומית + "ייצוא JSON" + דחיפה ידנית לקוד זה דפוס עבודה סביר ופשוט לניהול.
אם תרצה בהמשך שעריכת נכסים תסתנכרן אוטומטית בכל המכשירים — אפשר להרחיב את ה-Apps Script
עם `properties-overrides` (זה עוד ~100 שורות קוד). פשוט תגיד.*

## עדכון הגדרות אחרי הפריסה הראשונה

אם שינית משהו ב-`CONFIG` ב-`SHIBUMI_backend.gs` (למשל את המייל או ה-Token):

1. שמור (Ctrl+S).
2. **Deploy → Manage deployments** → לחץ על העיפרון (✏) ליד הפריסה הקיימת.
3. **Version: New version** → **Deploy**.
4. ה-URL נשאר אותו דבר — לא צריך לעדכן את האתר.

## פתרון בעיות

**בעיה:** בטופס יצירת הקשר רואים "פרטיכם נשמרו אצלנו, אם דחוף..." במקום "הפנייה התקבלה".
**פתרון:** ה-Backend לא מגיב. פתח את Console בדפדפן (F12) וחפש שגיאות.
בדרך כלל — `BACKEND_URL` ריק או שגוי ב-`config.js`.

**בעיה:** במערכת הניהול, שורת הסטטוס אומרת "● השרת לא זמין".
**פתרון:** ה-`ADMIN_TOKEN` ב-`config.js` לא תואם ל-`CONFIG.ADMIN_TOKEN` ב-Apps Script.

**בעיה:** לא מגיע מייל.
**פתרון:** בדוק ב-Google Apps Script → Executions → אם יש כשל ב-`mailBroker_`.
ייתכן שחצית את מכסת המייל היומית של Apps Script (100/יום לחשבון חינמי — יותר מספיק).
